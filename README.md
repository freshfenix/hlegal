# hlegal — Юридична компанія ⚖️

## Фінальна випускна робота

---

### 📌 Про проєкт

**hlegal** — це сучасний, високопродуктивний та повністю адаптивний вебсайт для преміальної юридичної компанії. Даний проєкт є фінальною випускною роботою, виконаною з дотриманням найсуворіших і найсучасніших стандартів веб-розробки:

- **Семантична та валідна верстка:** Використання сучасних стандартів HTML5 для забезпечення SEO-оптимізації та доступності (Accessibility).
- **Компонентна архітектура стилів:** Організація стильового коду за допомогою CSS-препроцесора **SCSS** та методології BEM, що робить кодову базу легкою для розширення та супроводу.
- **Сучасна збірка проєкту:** Використання швидкого інструменту збірки **Vite 6** для мінімізації та оптимізації результуючих файлів.
- **Responsive Web Design (RWD):** Повна адаптивність під усі види пристроїв (від мобільних телефонів з шириною екрану 320px до широкоформатних моніторів) з плавними CSS-анімаціями.
- **Оптимізація продуктивності:** Автоматична оптимізація та стиснення зображень, генерація форматів нового покоління (**WebP**) та інтеграція SVG-спрайтів.

---

### 🚀 Технологічний стек (Tech Stack)

Проєкт розроблено із застосуванням сучасних інструментів та бібліотек:

- **Core:** HTML5 (семантична розмітка), CSS3 (SCSS / Sass), Vanilla JavaScript (ES6+).
- **Складальник (Bundler):** Vite 6.x — забезпечує надшвидкий Hot Module Replacement (HMR) під час розробки та оптимізовану збірку для продакшену.
- **Бібліотеки та плагіни:**
  - [Swiper.js](https://swiperjs.com/) (v12) — для розробки гнучких та адаптивних слайдерів (відгуки клієнтів, досягнення компанії).
  - [Vite Image Optimizer](https://github.com/FatehG/vite-plugin-image-optimizer) — плагін для автоматичного стиснення векторної та растрової графіки.
  - `imagemin` & `imagemin-webp` — плагіни для автоматичної конвертації зображень у надлегкий формат WebP.
  - `PostCSS Autoprefixer` — додавання кросбраузерних префіксів до CSS-стилів.
  - `PostCSS Sort Media Queries` — сортування та об'єднання `@media` запитів для зменшення фінального розміру файлу стилів.

### 💻 Інструкція з локального розгортання (Installation & Setup)

Виконайте наступні кроки, щоб запустити проєкт локально на вашому комп'ютері:

#### 1. Передумови (Prerequisites)

Переконайтеся, що на вашому комп'ютері встановлено **Node.js** версії 18.0.0 або вище. Перевірити версію можна командою:

```bash
node -v
```

#### 2. Клонування репозиторію

Схилюйте проєкт або завантажте та розархівуйте його локально:

```bash
git clone https://github.com/freshfenix/hlegal.git
cd hlegal
```

#### 3. Встановлення залежностей

Запустіть команду для автоматичного встановлення всіх модулів:

```bash
npm install
```

#### 4. Запуск сервера розробки (Development Mode)

Запустіть локальний сервер із миттєвим оновленням сторінок (HMR):

```bash
npm run dev
```

Після успішного запуску термінал покаже посилання. Зазвичай це `http://localhost:5173`. Відкрийте його в браузері.

#### 5. Збірка для продакшену (Production Build)

Для створення фінальної, оптимізованої та стиснутої версії сайту виконайте:

```bash
npm run build
```

Всі файли збірки будуть автоматично оптимізовані (включаючи конвертацію та стиснення зображень) і збережені в директорію `dist/`.

#### 6. Попередній перегляд збірки

Щоб перевірити, як працює зібрана продакшен-версія локально:

```bash
npm run preview
```

### ⚙️ Ключові функціональні модулі та UI-рішення

- **Адаптивне меню навігації (Burger Menu):**
  Реалізовано за допомогою JavaScript та CSS-анімацій. Забезпечує зручний доступ до всіх розділів сайту на смартфонах. Доступні як світла, так і темна теми шапки для різних сторінок сайту.
- **Інтерактивна панель цінностей (Dynamic Tabs):**
  На головній сторінці працює компонент вкладок, який дозволяє користувачеві переглядати філософію компанії («Consistent», «Customer oriented», «Innovative», «Business oriented») без перезавантаження сторінки.
- **Каруселі та слайдери (Swiper Integration):**
  - Слайдер відгуків на головній сторінці з автоматичною ротацією (autoplay) та точками пагінації.
  - Адаптивний слайдер досягнень на сторінці «Про нас», який перемикає сітку елементів залежно від ширини екрану пристрою.
- **Система іконок SVG-Sprite:**
  Всі іконки підключаються через загальний файл `sprites.svg` та тег `<use>`, що усуває зайві HTTP-запити до сервера й прискорює відображення сторінок.

### 📈 Оптимізація та продуктивність (Web Performance)

Проєкт пройшов комплексне налаштування для отримання максимальних балів у тестах Lighthouse:

1. **Зображення нового покоління:** Завдяки плагіну `imagemin-webp` під час розробки створюються та використовуються легкі зображення формату `.webp`.
2. **Lossless-стиснення:** Растрові та векторні графічні файли стискаються плагіном `ViteImageOptimizer` із балансом якості 70%, зменшуючи вагу сторінок у декілька разів.
3. **Об'єднання медіа-запитів:** Дозволяє оптимізувати вихідний CSS-код шляхом об'єднання однакових `@media` блоків.
4. **Доступність (a11y):** Всі інтерактивні елементи мають описові атрибути `aria-label` для екранних зчитувачів.
